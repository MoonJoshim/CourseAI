# 개발 동기 및 목적

- **교육 정보 접근성 향상**: 학과별 개설과목 현황과 세부 정보를 한눈에 확인할 수 있는 통합 플랫폼이 부족해, 학생들이 수강 계획을 세우기가 어려웠습니다. 본 프로젝트는 소프트웨어학과 학생들이 신뢰도 높은 정보를 빠르게 확보하도록 지원하는 것을 최우선 목표로 삼았습니다.
- **AI 기반 학습 지원 수요**: 수강신청과 진로 설계를 돕는 지능형 도구에 대한 요구가 증가했습니다. 검색, 추천, 질의응답 등 다양한 학습 보조 기능을 한곳에 모아 학습 경험을 고도화하려는 의도가 있었습니다.
- **운영 효율성 제고**: 기존에는 엑셀·크롤링 스크립트·DB 관리 등이 분산되어 있어 유지보수가 어렵고 오류 발생 가능성이 컸습니다. 이를 표준화된 백엔드/프론트엔드 구조로 정리해 운영 부담을 줄이고자 했습니다.

# 주요 기술

- **프론트엔드**: React(Create React App), React Router, Tailwind CSS, PostCSS, lucide-react 아이콘. 정적 JSON 데이터를 활용해 개설과목을 렌더링하고, AI 챗 인터페이스 등 다양한 페이지 컴포넌트를 구성했습니다.
- **백엔드**: Flask로 구성된 REST API(`lecture_api.py`, `ai_api.py`). `pymongo`를 통한 MongoDB 연동, `pandas`를 이용한 엑셀 데이터 처리, `python-dotenv` 기반 환경 변수 관리 구조를 채택했습니다.
- **데이터 수집 및 전처리**: Selenium과 `webdriver-manager`로 강의 정보를 크롤링하고, `scripts/export_software_courses.py`에서 소프트웨어학과 강의만 추출해 JSON으로 변환했습니다.
- **AI 및 RAG**: `google-generativeai`, `openai`, `pinecone-client`, `sentence-transformers`를 활용한 RAG 기반 질의응답. Pinecone 벡터 스토어와 임베딩 모델(`intfloat/multilingual-e5-base`)로 학습 자료를 색인했습니다.
- **인프라 및 운영**: 가상환경(venv)로 의존성을 분리하고, Flask + Gunicorn 배포를 전제. MongoDB는 로컬/원격 구성 모두 고려했으며, 개발 중 `setupProxy.js`를 통해 API 프록시를 구성했다가 요구에 따라 제거했습니다.

# 개발 내용

- **개설과목 현황 페이지 개선**: 초기에는 백엔드 API를 통해 MongoDB 데이터를 가져왔으나, DB 인증 오류 및 프록시 문제로 인해 프론트엔드에서 직접 `softwareCourses.json`을 로드하는 방식으로 전환했습니다. 검색·페이징 로직을 프론트에서 처리하도록 수정해 안정적인 표시를 확보했습니다.
- **데이터 파이프라인 정비**: `load_software_courses_from_excel()` 함수를 통해 엑셀 파일에서 강의를 필터링하고 캐싱하도록 백엔드를 보강했습니다. 스크립트 실행 한 번으로 최신 데이터를 JSON에 반영할 수 있도록 흐름을 단순화했습니다.
- **AI 기능 강화**: `/api/chat`, `/api/rag/chat` 엔드포인트를 구성해 일반 대화와 RAG 대화 모드를 지원했습니다. 환경 변수로 LLM 제공자, API 키, 백엔드 URL 등을 분리해 배포 환경에 맞춰 손쉽게 조정할 수 있도록 했습니다.
- **UI/UX 정리**: 네비게이션 바에서 사용하지 않는 트렌드 분석·맞춤 추천 메뉴를 제거하고, 실제 제공 중인 기능 위주로 화면을 구성했습니다. Tailwind를 이용해 반응형 레이아웃과 일관된 스타일을 유지했습니다.
- **환경 셋업 가이드 작성**: Virtualenv 재정비, Python 3.11 업그레이드, NumPy 호환성 문제 대응(pip 재설치, `numpy<2`로 다운그레이드) 등 개발 환경 구축 절차를 명문화했습니다. 또한 `.env.example`를 통해 프론트엔드 환경 변수를 문서화했습니다.

# 결과 및 분석

- **가시적 성과**: 프론트엔드에서 소프트웨어학과 강의 정보를 안정적으로 표시할 수 있게 되었고, 사용자는 DB 상태와 무관하게 즉시 정보를 확인할 수 있습니다. API 프록시 제거 후에도 기능이 정상 동작하도록 구조가 단순화되었습니다.
- **AI 워크플로우 준비**: Pinecone, SentenceTransformer 등 고각성 의존성을 설치하고 오류를 해결해 RAG 파이프라인을 실행할 토대를 마련했습니다. 다만 NumPy와 OpenSSL 버전 불일치 등 macOS 특유의 호환성 문제가 빈번했으며, 장기적으로는 Docker 기반 표준 환경을 마련할 필요가 있습니다.
- **유지보수성 향상**: 프론트엔드와 백엔드에서 사용하는 환경 변수를 분리해 설정 충돌을 최소화했습니다. 기능별 파일 구조가 정리되어 향후 기능 추가(예: 맞춤 추천 재도입, 트렌드 분석 복구)도 용이한 상태입니다.
- **향후 과제**:
  - 엑셀 → JSON 수동 변환 과정을 CI/CD 파이프라인에 연결해 자동화.
  - MongoDB 인증 문제를 근본적으로 해결해 실시간 DB 연동을 복구하거나, 서버리스 DB 서비스로 이전 검토.
  - RAG 서비스 운영 시 모델 로딩 및 추론 비용을 고려한 캐싱 전략 도입, 지표 기반의 성능 모니터링 체계 구축.
  - 사용자 피드백을 반영한 UI 개선(검색 필터 다양화, 북마크/시간표 도구 등) 및 접근성 강화.
